Tutorial membuat Todo list dengan alpine js

#Alpine js#JavascriptAlfarizi · 05 Dec 2022

Setelah berkenalan dengan apa itu Alpine js disini,kali ini penulis akan memberikan tutorial membuat project menggunakan Alpine js, yaitu membuat aplikasi todo list sederhana.

Sebelum mengikuti tutorial yaitu:

  1. Browser(jangan yang terlalu jadul, alpine js bisa tidak bekerja)
  2. Text editor(vscode,sublime,notepadd++ dll)
  3. Jaringan internet untuk mendownload alpine js, tapi jika menyimpan alpine js secara offline, maka tidak diperlukan internet

Untuk fitur fitur yang ada pada todo list yang akan dibuat adalah sebagai berikut:

  1. Menambahkan item
  2. Menghapus item
  3. Menandai item yang telah selesai

Terlihat sulit? tapi karena kita menggunakan alpine js kita dapat membuatnya dengan mudah. Sekarang mari lanjut ke cara pembuatannya.

Membuat struktur dasar file html

Pastinya sebelum membuat webisite kita harus membuat html beserta struktur dasarnya. Berikut adalah strukturnya:

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Document</title>
    </head>
    <body>

    </body>
</html>

Jika di text editor kalian terinstall plugin emmet, struktur html di munculkan dengan mengetikan ! lalu tekan Tab.

Jika sudah, simpan file htmlnya dengan nama apapun yang berakhiran .html

Menginstall alpine js

Untuk instalasi alpine js ada 2 cara yaitu menggunakan link cdn dan mendownloadnya untuk menggunkannya secara offline.

  • Installasi menggunakan link dari cdn

Masukan tag berikut ke dalam bagian head dari html: 

<script src="https://unpkg.com/alpinejs" defer></script>
  • Menggunakan secara offline

Pergi ke link ini, kemudian copy semua kontennya lalu paste-kan ke dalam file javascript, simpan dengan nama: alpine.js. Untuk tempat menyimpannya penulis sarankan di folder yang sama dengan file html tadi agar lebih mudah ditemukan.

Struktur folder jika menyimpan alpine js secara offline:

Setelah itu masukan tag berikut ke dalam bagian head dari html: 

<script src="alpine.js" defer></script>

Untuk mengecek kita bisa membuat program sederhana dengan alpine js, yaitu sebagai berikut: 

<div x-data>        
    <span x-text="'hello world'"></span>
</div>

Masukan tag tersebut ke dalam body dari html, lalu lihat apakah ada tulisan 'hello world' yang muncul? Perlu diingat jika alpine js akan menganggap setiap nilai di dalam atribut khususnya sebagai kode javascript, maka jika tidak ada teks yang tampil, mungkin kutip satunya lupa ditambahkan.

Sederhananya seperti ini:

❌ Tidak akan muncul apa apa karena nilai atribut adalah kode javascript yang salah(tidak ada kutip satu).

<span x-text="hello world"></span>

✅ Teks akan muncul, karena penulisan string pada javascript benar(ada kutip satu).

<span x-text="'hello world'"></span>

Dan perlu diperhatikan juga jika setiap nilai string pada javascript di alpine js, tidak boleh menggunakan kutip dua("") ,sebagai gantinya gunakan kutip satu('') atau simbol backtick(``).

Membuat komponen dasar todo list

Jika semua sudah terinstall, kosongkan bagian body dari html lalu buat komponen alpine js.

<div x-data="{todos: [],todoName: ''}" >

</div>

Nah di dalam komponen tersebut ada dua variabel, yaitu todos dan todoName. Variabel todos ini akan digunakan untuk menyimpan kumpulan item yang bertipe array sedangkan todoName akan menyimpan nilai string dari data yang di input sebelum dimasukan ke array.

dan jika digabung dengan html tadi maka akan menjadi:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <script src="alpine.js" defer></script>
    <title>Document</title>
</head>
<body>
    <div x-data="{todos: [],todoName: ''}">

    </div>
</body>
</html>

Saat ini file html masih kosong dan belum ada yang tampil.

Membuat input dan memasukan item

Nah jika sudah membuat komponen dan data awalan, buatlah elemen input di dalam komponen dengan atribut x-model.

<input type="text" x-model="todoName" placeholder="masukan item">

Nah x-model ini akan mengikat variabel todoName pada elemen input, jadi setiap kita melakukan perubahan pada input nilai variabel todoName juga akan ikut berubah.

Jika ingin mengecek jika nilai variabel berubah, kita bisa coba lihat dengan menambahkan elemen lain untuk testing di bawah input tadi.

<p>Nilai input: <span x-text="todoName"></span> </p>

Refresh browser kalian lalu coba masukan dan ubah nilai di input. Apakah seperti ini?

Jika sudah benar, hapus lagi elemen p yang tadi untuk testing, ganti dengan button, maka komponen nya akan tampak seperti ini: 

<div x-data="{todos: [],todoName: ''}">
    <input type="text" x-model="todoName" placeholder="masukan item">           
    <button>Tambah item</button>
</div>

button nya masih belum bisa melakukan perubahan terhadap data,agar button dapat melakukannya tambahkan atribut x-on:click yang berisi program yang dijalankan setiap button di klik.

<div x-data="{todos: [],todoName: ''}">
    <input type="text" x-model="todoName" placeholder="masukan item">           
    <button x-on:click="todos.push(todo);todoName=''">Tambah item</button>
</div>

todo.push digunakan untuk menambahkan item ke array lalu kode disebelah titik koma digunakan untuk membersihkan input. Nah tampilannya masih kosong karena memang tidak ada elemen yang dibuat untuk menampilkan data. Untuk menampilkannya gunakan kita dapat menggunakan x-for.

    <div x-data="{todos: [],todoName: ''}">
        <input type="text" x-model="todoName" placeholder="masukan item">           
        <button x-on:click="todos.push(todoName);todoName=''">Tambah item</button>
        <li>
            <template x-for="todo in todos">
                <ul>
                    <span x-text="todo"></span>
                </ul>
            </template>
        </li>
    </div>

Simpan lalu refresh lagi halaman html kalian. Maka tampilan akan menjadi seperti berikut:

Apakah sudah berhasil?

Nah agar lebih rapih daripada menggunakan button saja, mari kita ubah komponennya menjadi didalam form.

    <form x-on:submit.prevent="todos.push(todoName);todoName=''" x-data="{todos: [],todoName: ''}">
        <input type="text" x-model="todoName" required placeholder="masukan item">          
        <button type="submit">Tambah item</button>
        <li>
            <template x-for="todo in todos">
                <ul>
                    <span x-text="todo"></span>
                </ul>
            </template>
        </li>
    </form>

x-on:submit.prevent adalah event yang akan menjalankan kode di dalam atribut jika melakukan submit, dan prevent dibelakangnya untuk mencegah terjadi refresh saat submit. Jika menggunakan form user dapat melakukan input dengan tombol enter dan dapat mencegah user memasukan data kosong dengan atribut required pada elemen input.

Membuatnya lebih kompleks

Nah,setelah memastikan todo list dapat menambahkan item, mari kita tambahkan fitur lainnya sehingga tampilanya lebih baik.

Pertama tama ubah dulu data yang ditambahkan di x-on:submit.prevent menjadi object, sehingga kita dapat menyimpan lebih dari satu data pada sebuah elemen array.

...
<form x-on:submit.prevent="todos.push({id: todos.length,name: todoName, complete: false});todoName=''" x-data="{todos: [],todoName: ''}">
...

Didalam object yang ditambahkan akan ada:

  • id: akan digunakan sebagai pembeda dari setiap item yang nilainya diambil dari jumlah item pada array
  • name: adalah nama dari item tersebut
  • complete: adalah nilai yang menentukan jika item sudah selesai atau belum.

Kemudian elemen listnya diubah menjadi:

...
<li>
    <template x-for="todo in todos" :key="todo.id">
        <ul>
            <input type="checkbox">
            <span x-text="todo.name"></span>
            <button>Hapus</button>
        </ul>
    </template>
</li>
....

Karena sekarang bentuk item di dalam array adalah object, kita tidak bisa langsung mengakses nama dari item, dan harus menggunakan key dari object, seperti: todo.name,todo.id,todo.complete. Lalu untuk atribut :key digunakan untuk memastikan setiap item yang di loop memiliki nilai yang berbeda dari elemen lain sehingga memudahkan alpine js mendeteksi perubahan pada list.

Jika mengikuti langkah diatas kode dari komponen yang kita buat akan tampak seperti ini: 

<form x-on:submit.prevent="todos.push({id: todos.length,name: todoName,complete: false});todoName=''" x-data="{todos: [],todoName: ''}">
    <input type="text" x-model="todoName" required placeholder="masukan item">          
    <button type="submit">Tambah item</button>
    <li>
        <template x-for="todo in todos" :key="todo.id">
            <ul>
                <input type="checkbox">
                <span x-text="todo.name"></span>
                <button>Hapus</button>
            </ul>
        </template>
    </li>
</form>

Sekarang list item sudah memiliki checkbox dan button di sampingngnya, sekarang tambahkan fungsionalitas untuk keduanya.

Yang pertama checkbox dahulu, jadi saat checknya dirubah(checked/uncheked) maka teks akan di coret. Untuk melakukannya kita menggunakan atribut x-on:change artinya program akan di dijalankan jika nilai elemen berubah.

...
<input type="checkbox" x-on:change="todo.complete = !todo.complete">
<span x-text="todo.name"></span>
<button>Hapus</button>
...

Nah, jika di klik maka nilai atribut complete pada object dari todo akan berubah sesuai kebalikannya(jika false jadi true, true jadi false).

Karena sekarang atribut pada object todo sudah bisa berubah, sekarang kita tambahkan class css yang akan mencoret teks yang telah selesai. Pastinya buat css di dalam tag style di file yang sama.

<style>
    .complete{
        text-decoration: line-through;
    }
</style>

Sekarang css untuk menandai item sudah ada, lalu bagaimana cara menambahkan class-nya ke item di list? Cara nya mudah, alpine js memilki atribut khusus untuk tugas ini, yaitu x-bind: atau :<nama atribut>(ganti <nama atribut> dengan nama atribut di html, misalnya: :class,:style,:required dsb). Atribut ini memilki fungsi untuk menjadikan atribut pada html menjadi dinamis.

Sekarang kita tempatkan atribut :class(bentuk sederhana dari x-bind:class) pada teks untuk item nya(di span).

...
<span x-text="todo.name" :class="todo.complete?'complete':''"></span>
...

Kita menggunakan operator ternary pada javascript untuk menentukan kapan class ditambahkan.

Sekarang kita beralih ke fungsi untuk menghapus, caranya mudah, tinggal tambahkan x-on:click pada tombol hapus, lalu di dalamnya terdapat fungsi filter untuk menghapus salah satu item pada array. Bentuknya sebagai berikut: 

...
<button type="button" x-on:click="
        todos = todos.filter(function(NewTodo){
            return NewTodo.id != todo.id
        })
">Hapus</button>
...

Sekarang bentuk dari komponen todo list nya adalah: 

<style>
    .complete{
        text-decoration: line-through;
    }
</style>
<form x-on:submit.prevent="todos.push({id: todos.length,name: todoName,complete: false});todoName=''" x-data="{todos: [],todoName: ''}">
    <input type="text" x-model="todoName" required placeholder="masukan item">          
    <button type="submit">Tambah item</button>
    <li>
        <template x-for="todo in todos" :key="todo.id">
            <ul>
                <input type="checkbox" x-on:change="todo.complete = !todo.complete">
                <span x-text="todo.name" :class="todo.complete?'complete':''"></span>
                <button type="button" x-on:click="
                        todos = todos.filter(function(NewTodo){
                        return NewTodo.id != todo.id
                    })
                ">Hapus</button>
            </ul>
        </template>
    </li>
</form>
